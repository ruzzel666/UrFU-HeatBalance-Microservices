# Проект «Разработка микросервисов расчёта теплотехнических параметров различных типов сушильных печей»: Руководство по Frontend-разработке

Настоящий документ описывает архитектуру, стек технологий и пошаговый план разработки графического веб-интерфейса для системы расчёта теплотехнических параметров сушильных печей.

---

## 1. Архитектура и роль Frontend-приложения
Frontend-приложение (SPA) выступает в качестве единого пользовательского интерфейса для взаимодействия с микросервисным бэкендом (Java/Spring Boot). 

**Основные задачи интерфейса:**
1. Ввод и строгая валидация технологических параметров (более 30 параметров для камерной печи).
2. Визуализация результатов расчёта теплового баланса (таблицы, графики, расчёт КПД).
3. Обеспечение безопасности посредством передачи OAuth 2.0 токенов (Keycloak) к API.

## 2. Утвержденный стек технологий
Для обеспечения высокой производительности и профессионального внешнего вида используется следующий стек:

* **Инструмент сборки:** Vite (быстрая сборка и HMR).
* **UI Библиотека:** Ant Design (antd) — готовые компоненты форм, таблиц и валидации.
* **Фреймворк:** React (функциональные компоненты, Hooks).
* **Маршрутизация:** React Router v6 (навигация между типами печей).
* **Сетевые запросы:** Axios (с настроенными интерцепторами для JWT-токенов).
* **Инфраструктура:** Docker (многоэтапная сборка с Nginx) и Jenkins для CI/CD.

## 4. План разработки (Roadmap по итерациям)

### Итерация 1: Инициализация
* Развернуть проект через `npm create vite@latest`.
* Установить зависимости: `antd`, `react-router-dom`, `axios`.
* Настроить базовый `docker-compose.yml` (в корне) для запуска Keycloak.

### Итерация 2: Проектирование UI и форм
* Создать `MainLayout` с использованием `<Layout>` и `<Sider>` из Ant Design.
* Сверстать форму для "Камерной печи" с использованием `<Form>` и `<InputNumber>`. 
* **Правило:** Группировать 33 параметра по логическим блокам (Характеристики материала, Топливо, Геометрия) через `<Tabs>` или `<Collapse>`.

### Итерация 3: Логика и Аутентификация
* Настроить `axiosInstance.js` с добавлением `Authorization: Bearer <token>` в заголовки.
* Подключить мок-запросы для тестирования форм до готовности Java-бэкенда.
* Сверстать вывод результатов теплового баланса с помощью `<Table>` (статьи прихода/расхода) и `<Progress type="circle">` (для вывода КПД).

### Итерация 4: Интеграция с Backend (Java)
* Заменить мок-данные на реальные REST-запросы к Java-микросервисам.
* Настроить поддержку форматов JSON/XML в заголовках (`Accept`).
* Провести сквозное E2E-тестирование на контрольных примерах (например, масса 10 000 кг, влажность 8%).
* Обработать ошибки сервера через `message.error()` из Ant Design.

### Итерация 5: Контейнеризация (Docker)
* Написать `Dockerfile` (Multi-stage: этап `build` на Node.js, этап `production` на Nginx).
* Настроить файл `nginx.conf`, чтобы все маршруты вели на `index.html` (решение проблемы 404 при обновлении страницы).
* Обновить корневой `docker-compose.yml`, добавив запуск собранного образа фронтенда.

### Итерация 6: CI/CD и Релиз
* Написать `Jenkinsfile` для автоматической сборки Docker-образа при push-событиях в ветку `main`.
* Завершить полировку UI (адаптивность, читаемость шрифтов).
* Подготовить скриншоты интерфейса для итогового отчета и "Руководства пользователя".

## 5. Требования к качеству кода и UX
1. **Строгая валидация:** Пользователь не должен иметь возможности отправить форму с отрицательной массой или пустыми обязательными полями (использовать `rules` в `<Form.Item>`).
2. **Обратная связь:** Всегда показывать состояние загрузки (`loading={true}`) на кнопках при ожидании ответа от API.
3. **Форматирование:** Выводить финансовые и физические величины с точностью до 2-х знаков после запятой, четко указывать единицы измерения (кг, м³, кДж) в лейблах форм.
