# Руководство администратора

## 1. Назначение

Администратор (уполномоченный пользователь) выполняет сопровождение данных через веб‑приложение:

- просмотр наборов исходных данных пользователей
- удаление некорректных/дублирующихся наборов

## 2. Первичная настройка

1) Укажите строку подключения к PostgreSQL в `ProductWebApp/appsettings.json`:

- `ConnectionStrings:DefaultConnection`

2) Укажите учётные данные администратора:

- `Seed:AdminEmail`
- `Seed:AdminPassword`

При запуске приложения администратор будет создан автоматически (если отсутствует) и добавлен в роль `Admin`.

## 3. Миграции БД

Из корня решения:

```bash
dotnet tool restore
dotnet tool run dotnet-ef database update --project "ProductWebApp\ProductWebApp.csproj" --startup-project "ProductWebApp\ProductWebApp.csproj"
```

## 4. Административный раздел

Войдите под администратором и откройте:

- `/Admin/DataSets`

В разделе доступна таблица наборов исходных данных всех пользователей и действие **удаления**.

## 5. Резервное копирование

Рекомендуется выполнять регулярные бэкапы PostgreSQL средствами `pg_dump`/`pg_dumpall` согласно принятому регламенту.
