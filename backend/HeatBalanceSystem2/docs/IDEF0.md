# Функциональная модель ИС (IDEF0)

Ниже приведено текстовое описание IDEF0‑модели (диаграммы A‑0/A0 и декомпозиция функций). Графические диаграммы можно построить в любой IDEF0‑совместимой нотации (Visio/Draw.io/IDEF0‑редактор), используя те же ICOM‑потоки.

## Диаграмма контекста (A‑0) «Расчёт теплового баланса сушильных установок»

**Функция**: обеспечить авторизованным пользователям выполнение расчётов теплового баланса и выпуск отчётов с хранением исходных данных в БД.

### ICOM (A‑0)

- **Входы (Input)**:
  - Набор исходных данных пользователя (параметры материала/режима/энергоносителя)
  - Комментарии пользователя к наборам данных

- **Управление (Control)**:
  - Методики/нормативы расчёта теплового баланса (формулы, допущения, единицы измерения)
  - Политики безопасности (аутентификация/авторизация, роли)
  - Требования к отчётам (форма, обязательные показатели)

- **Выходы (Output)**:
  - Результаты расчёта (численные показатели)
  - Графическое представление статей баланса
  - Отчёт (экран + экспорт PDF)

- **Механизмы (Mechanism)**:
  - Веб‑приложение ASP.NET Core (UI + бизнес‑логика)
  - База данных PostgreSQL
  - Математические библиотеки расчёта (C#)
  - Средства генерации PDF
  - Пользователи и администратор системы

## Диаграмма A0 (декомпозиция верхнего уровня)

A0 «Система расчёта теплового баланса» декомпозируется на функции:

- **A1 Управление пользователями** (регистрация/вход/выход/личные данные)
- **A2 Управление наборами исходных данных** (создание/изменение/сохранение/удаление)
- **A3 Выполнение расчётов** (запуск расчёта по выбранному набору, сохранение результатов)
- **A4 Представление и визуализация результатов** (таблицы, графики)
- **A5 Формирование отчётов** (просмотр на экране, экспорт в PDF)
- **A6 Администрирование** (сопровождение данных уполномоченным пользователем)

### Связи между функциями (пример)

- A1 предоставляет контекст «текущий пользователь/роль» для A2–A6.
- A2 формирует и хранит «набор исходных данных», который является входом A3.
- A3 формирует «результат расчёта», который используется A4 и A5.
- A6 управляет данными (просмотр/удаление наборов) в рамках политики безопасности.

## Детализация функций (кратко)

### A1 Управление пользователями

- **Вход**: регистрационные данные, учётные данные для входа
- **Управление**: политики паролей/ролей, правила уникальности email
- **Выход**: учётная запись, сессия пользователя
- **Механизм**: ASP.NET Core Identity + PostgreSQL

### A2 Управление наборами исходных данных

- **Вход**: параметры для выбранного типа сушильной установки, комментарий
- **Управление**: схема данных, валидация диапазонов
- **Выход**: сохранённый набор данных (в БД)
- **Механизм**: UI (Razor Pages) + EF Core

### A3 Выполнение расчётов

- **Вход**: выбранный набор данных
- **Управление**: методика расчёта (формулы/допущения)
- **Выход**: вычисленные статьи теплового баланса и KPI (КПД и др.)
- **Механизм**: библиотека `HeatBalance.Math`

### A4 Представление и визуализация результатов

- **Вход**: результаты A3
- **Управление**: требования к визуализации
- **Выход**: таблицы, график статей баланса
- **Механизм**: Chart.js в UI

### A5 Формирование отчётов

- **Вход**: набор данных + результат расчёта
- **Управление**: шаблон отчёта
- **Выход**: PDF‑файл и экранная форма отчёта
- **Механизм**: QuestPDF

### A6 Администрирование

- **Вход**: данные пользователей (наборы/результаты)
- **Управление**: роль Admin, регламент сопровождения
- **Выход**: изменения в данных (удаление/корректировка)
- **Механизм**: административные страницы + ограничения доступа
